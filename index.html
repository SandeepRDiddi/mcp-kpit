Slide 02 · The Problem
Why AI fails without MCP

AI models are trained on vast knowledge — but that knowledge goes stale. Without live connections, every answer about your system is a guess.

📅
Time-blind by default
Training data has a cutoff. Anything after that date is invisible to the model.
🔒
No access to your private data
Your databases, internal files, customer records — the AI has never seen any of it.
🔧
Custom integrations don't scale
Before MCP, every team built one-off API wrappers. Fragile. Expensive. Not reusable across AI tools.
🎲
Hallucination fills the gap
When the AI can't look something up, it generates a plausible-sounding answer. Confidently wrong.

Slide 04 · Architecture
How MCP works

Three components. One clean flow. Zero guesswork.

1
MCP Host — the AI application
Claude Desktop, your custom chatbot, Copilot Studio. Runs the conversation, decides when to call external tools.
2
MCP Client — inside the Host
Manages secure connections to each MCP Server. Sends requests, handles auth, returns structured responses.
3
MCP Server — your data service
A small service you build that exposes your data and tools through a standard interface. One server can expose a database, calendar, file system — or a whole mall system.
The flow: User asks → Host decides what to look up → Client calls Server → Server fetches real data → Host replies with an accurate, live answer.
📦
Slide 05 · Capabilities
What an MCP Server exposes

Three capability types — any server can offer any combination.

🛠️
Tools
Functions the AI can call — with typed inputs so the AI knows exactly how to use them.
run_query() · book_ticket()
📄
Resources
Structured data identified by URI — files, records, documents — read on demand.
menu.json · profile/{id}
💬
Prompts
Reusable prompt templates — standardised workflows like "summarise this order" or "generate a release note from these commits".

Slide 08 · Quick Start
Get started in 3 steps

Your first working MCP server in under a day — no boilerplate required.

1
Install the SDK
# Python
pip install mcp

# Node.js
npm install @modelcontextprotocol/sdk
2
Define your tools
from mcp.server import Server
app = Server("my-server")

@app.tool()
async def check_stock(item: str):
  """Check if item is in stock"""
  return db.query("SELECT ...")
3
Connect to your AI
Register the server in Claude Desktop, LangGraph, or any MCP-compatible AI. The AI automatically discovers your tools and calls them when relevant. First live query in ~30 minutes.
